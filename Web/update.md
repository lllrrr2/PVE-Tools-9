# 更新日志（PVE Tools Pro）

> 本页用于同步脚本正式发布版本。v7.0.0 为一次覆盖 CLI、安全边界、虚拟机运维、宿主机网络与 GPU 直通体系的史诗级重构。

## 2026年5月15日 - v8.8.8
### Shell 版最终版本，项目归档并转向 Go 重构

- Shell 版本定格为最终版本，`main` 分支停止功能更新，后续维护将转向 Go 语言重构版（`beta-go` 分支）。
- 文档站首页新增 EOL 归档公告横幅，引导用户了解项目后续方向。
- 导航栏新增"归档说明"入口，集中展示项目归档背景与后续规划。
- 公告组件更新为项目归档声明，明确说明 Shell 版停止更新及 Go 重构版的开源地址。
- 注释掉 HarmonyOS Sans 外部字体引用，改用系统默认字体栈，减少外部依赖。
- 页脚版权信息更新为 Ciriu Networks 标识。
- 赞助数据同步更新。

#### 升级提醒
- Shell 版 v8.8.8 为最终版本，不再接收新功能更新。
- 如需继续使用 PVE Tools Pro，请关注 Go 重构版：[beta-go 分支](https://github.com/Mapleawaa/PVE-Tools-9/tree/beta-go)。
- 现有 Shell 版功能仍可正常使用，但不再提供新特性支持。

## 2026年5月9日 - v7.7.0
### 修复阿里云换源 Ceph 镜像 404 问题，优化换源日志提示

- 修复阿里云换源时 Ceph 镜像指向不存在地址（`https://mirrors.aliyun.com/ceph/debian-squid` 返回 404）的 BUG，改为使用官方 Ceph 源。
- 优化换源功能的日志提示，明确告知用户哪些源使用了镜像站、哪些沿用官方源，避免混淆。
- 经验证，阿里云和腾讯云目前均不提供 PVE / Ceph / CT 模板镜像，脚本在选择这两家镜像站时会自动回退到官方源。

#### 升级提醒
- 如果你之前选择了阿里云换源并发现 Ceph 源更新失败，升级后重新执行一次换源操作即可修复。
- 阿里云和腾讯云目前仅支持 Debian 基础源和安全更新源的镜像加速，PVE / Ceph / CT 组件仍使用官方源。

## 2026年5月7日 - v7.6.0
### MAC 绑定网卡改用 systemd .link 方案，并补充腾讯云 / 阿里云公网换源选项

- 将按 MAC 地址绑定网卡到 Bridge 的实现改为 `systemd .link` 规则，生成 `/etc/systemd/network/10-pve-tools-*.link`，不再依赖老旧的 udev 持久化绑定方式。
- 新增绑定前冲突检测与宿主机网络引用检查，只有在确认不会影响现有网络拓扑时才写入配置，降低误绑和断网风险。
- 保留旧式 `/etc/udev/rules.d/70-persistent-net.rules` 的兼容清理逻辑，方便从历史方案平滑迁移到新实现。
- 换源菜单新增腾讯云公网镜像和阿里云公网镜像选项，其中腾讯云用于 Debian / 安全更新，阿里云用于 Debian / 安全 / Ceph。
- 脚本版本保持为 `7.6.0`，本次仅做同版本补充，不额外迭代。

#### 升级提醒
- 如果你之前已经手动写过旧式 udev 规则，这次更新后建议重新检查一次网卡和 Bridge 的绑定结果。
- 新实现会优先走 `systemd .link`，旧规则只负责兼容清理，不再作为主路径。
- 新增的公网镜像选项只会接入已确认可用的组件，避免把未验证的地址写入系统配置。

## 2026年5月5日 - v7.5.0
### Copy Fail 入口改为修复复查与清理，适配已陆续修复后的内核管理流程

- 将 `Copy Fail（CVE-2026-31431）` 入口语义从“检测/缓解”调整为“修复复查 / 清理 / 升级后复验”，更适合漏洞已逐步修复完成后的场景。
- 在主菜单和内核菜单增加修复后提示，提醒用户完成内核升级后尽快回到该入口检查状态、清理历史临时阻断配置并再次验证。
- 保留检测、手动说明、自动临时阻断、回滚和同步内核功能，但统一改写为“修复后管理”视角，减少误导用户把临时阻断当成长期方案。

#### 升级提醒
- 如果你已经完成内核更新，建议先进入 Copy Fail 菜单执行一次检测，再确认是否需要清理 `/etc/modprobe.d/disable-algif.conf` 和 `/etc/modprobe.d/disable-authencesn.conf`。
- 这次更新只调整入口语义和提示，不改变既有检测与内核同步逻辑。

## 2026年4月30日 - v7.4.0
### 内核更新修复：避免误装辅助包，统一真实内核包识别

- 修复自动更新流程误把辅助包当作最新内核的问题，避免再次出现 `proxmox-kernel-libc-dev` 之类的错误安装目标。
- 统一 `proxmox-kernel-*` / `pve-kernel-*` / `-signed` 的包名识别与规范化逻辑，确保菜单展示、自动更新和手动安装走同一套真实内核路径。
- 调整可用内核列表筛选规则，只保留真实内核包，减少混入开发包、工具包或其他辅助条目的风险。
- 保留 `Copy Fail（CVE-2026-31431）` 安全处置入口，并同步将脚本版本提升到 `7.4.0`。

#### 升级提醒
- 这次更新重点是修复内核管理链路，不影响已有的 Copy Fail 检测、缓解和回滚功能。
- 如果你之前误触发过内核更新失败，建议先重新打开“查看可用内核列表”确认包名，再执行安装或自动同步。

## 2026年4月30日 - v7.3.0
### 内核安全响应：新增 Copy Fail（CVE-2026-31431）检测、风险评估与缓解入口

- 新增 Copy Fail / `CVE-2026-31431` 漏洞处置菜单，挂载到“启动与内核 -> 内核管理”路径，便于围绕当前运行内核直接排查与处理。
- 新增漏洞状态检测：综合当前运行内核版本、已安装内核包 changelog、`CONFIG_CRYPTO_USER_API_AEAD` / `CONFIG_CRYPTO_AUTHENCESN`、`algif_aead` / `authencesn` 模块状态，判断系统更接近“已修复 / 已缓解 / 仍有风险 / 通常不受影响”哪一类。
- 新增受攻击危险评估：结合交互用户数量、LXC 容器数量与运行状态、SSH 服务暴露情况，对本地提权 / 容器逃逸前置条件做更直观提示。
- 新增手动处理说明：在界面中明确区分“正式修复优先”和“临时缓解优先”，提醒用户优先升级到供应商明确声明已修复该 CVE 的内核。
- 新增两种自动临时缓解方式：
  - 禁用 `algif_aead`（参考 copy.fail 官方建议）
  - 禁用 `authencesn`（参考发行版 workaround 思路）
- 新增临时缓解回滚入口，便于在完成正式内核修复后撤销持久化模块阻断策略。
- 保留并复用现有内核同步更新流程，但在交互文案中明确说明“升级到仓库最新内核”不自动等于“已经修复该 CVE”，避免误判风险边界。

#### 升级提醒
- 该版本重点是安全检测与缓解辅助，不包含 exploit、PoC 或自动化高风险利用逻辑。
- 若当前内核供应商尚未发布明确修复包，建议先启用临时缓解，再安排维护窗口升级并重启到修复后的内核。

## 2026年4月6日 - v7.0.0
### 史诗级重构：安全边界、VM 运维、宿主机网络与 GPU 直通能力全面升级

#### P0 / 最高优先级：数据安全边界与高风险操作防护体系重构
- 重构高风险数据操作确认链路：为备份、恢复、配置导入导出、模板、克隆、Cloud-Init、迁移、防火墙开关、宿主机网络等关键路径补齐更明确的风险说明与确认词机制。
- 强化脚本级免责声明与许可提示：首次进入脚本时明确提醒宿主机网络、桥接 / Bond / VLAN、防火墙，以及 VM、磁盘、快照、克隆、恢复、迁移等操作的破坏性后果。
- 修复多处高风险操作的安全边界问题，重点收紧 VM 定时备份、配置导入、宿主机网络提交、防火墙规则集导入、IPv6 NAT6 等路径的输入校验与回滚兜底。
- 接入误操作后数据恢复导向内容，强调“立刻停止写入、优先保护现场、必要时联系专业数据恢复”的顺序。

#### P1 / 高优先级：VM 生命周期运维能力大幅补全
- 新增 VM 备份与恢复工具链：支持 vzdump 备份、从备份恢复为新 VM、定时备份任务管理，并对压缩方式、模式、目标存储和保留策略做更严格校验。
- 新增 VM 配置导入 / 导出：支持结构化导出配置，并在导入失败时尽量自动清理半成品 VM 与回滚已重绑磁盘槽位。
- 新增模板 / 克隆 / Cloud-Init：支持模板转换、完整克隆、链接克隆、cloud image 导入、Cloud-Init 参数写入与模板化交付。
- 新增虚拟机磁盘管理、快照管理、启动顺序与网络管理、集群内迁移等高频运维路径。
- 新增 IMG 镜像导入路径与 VM / 容器定时开关机能力，完善日常交付与自动化运维闭环。

#### P1 / 高优先级：宿主机网络、防火墙、IPv6 与诊断工具箱上线
- 新增宿主机网络配置向导：支持 bridge、VLAN、Bond、IPv4 / IPv6 / SLAAC / DHCP、PVE 防火墙、安全组、IPv6 助手与网络诊断工具箱。
- 新增 staged 网络提交流程与失败自动回滚逻辑，降低误改 /etc/network/interfaces 导致 SSH / WebUI 断连的风险。
- 新增规则集导入 / 导出、安全组维护、防火墙多层级开关，以及 IPv6 透传 / NAT6 场景支持。
- 补齐 traceroute、mtr、nmap、tcpdump 与端口连通性检查，形成宿主机侧网络诊断全链路。

#### P1 / 高优先级：GPU / PCI 直通体系补强

- 官网新增或补充 VM 备份 / 迁移 / Cloud-Init、宿主机网络 / 防火墙 / IPv6、误操作后数据恢复等专题内容。
- 持续强化“先备份、再操作、最后验证”的运维顺序提示。

- 升级后建议优先检查：主菜单入口、VM 备份 / 恢复、Cloud-Init、宿主机网络 staged 提交、PVE 防火墙导入导出、AMD / NVIDIA / Intel 直通菜单。

- **新增引导配置辅助**：识别直通磁盘的 UEFI/Legacy 类型，给出 VM 固件类型、机器类型与启动顺序建议
